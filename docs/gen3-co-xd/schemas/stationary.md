# 固定シンボル生成

## 基本形

```
LID = getRand()
HID = getRand()
PID = LID | (HID << 16)
// *1
HAB = getRand()
// *2
SCD = getRand()

IVs = {
  H: HAB & 0x1F,
  A: (HAB >> 5) & 0x1F,
  B: (HAB >> 10) & 0x1F,
  C: (SCD >> 5) & 0x1F,
  D: (SCD >> 10) & 0x1F,
  S: SCD & 0x1F
}
```

## 割り込みによる乱数のズレ

個体生成の途中で描画の割込が発生することがある。
描画処理には乱数消費が1回含まれているため、生成される個体も、割込が入らなかった場合のものとは違うものになる。
俗に、割り込みの入っていないパターンは<strong>Method1</strong>、`*1`の位置で割込が発生するパターンは<strong>Method2</strong>、`*2`の位置で割り込みが発生するパターンは<strong>Method4</strong>と呼ばれる。

割込が入るのは、1フレームの間に個体生成が完了しなかったということであり、個体生成までの処理が重いほど
`Method1 < Method4 < Method2`
という風に生成パターンが変動する。
固定シンボルの場合は野生や孵化と異なり、個体生成までに重い処理が入ることが少ないため、ほとんどがMethod1で生成される。
しかし、ポケモンボックスでRSをプレイする場合など、ごく限られた状況においてはMethod4個体が生成されることが確認されている。

## フィールド特性
4世代以降とは異なり、3世代ではシンクロなどのフィールド特性が固定シンボルに対しては効果を発揮しない。
